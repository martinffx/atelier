# Research Checklist

Use this checklist to ensure thorough research before planning.

## Pre-Research

- [ ] Read the feature request or problem statement
- [ ] Understand the user stories and acceptance criteria
- [ ] Know what "done" looks like

## Codebase Exploration

- [ ] Find existing components in the same domain
- [ ] Review similar features for patterns
- [ ] Check for existing utilities or helpers
- [ ] Identify the data model and entities involved
- [ ] Find tests that illustrate expected behavior

## Technical Context

- [ ] Understand the current architecture
- [ ] Identify integration points (APIs, events, database)
- [ ] Know the dependency versions in use
- [ ] Check for existing patterns and conventions

## Risk Identification

- [ ] What are the unknown unknowns?
- [ ] What complexity is hidden?
- [ ] What could go wrong?
- [ ] What assumptions are we making?

## External Research

- [ ] Check documentation for new dependencies
- [ ] Look for best practices from library authors
- [ ] Search for common pitfalls
- [ ] Verify approach is feasible

## Output

- [ ] Research summary written
- [ ] Key findings documented
- [ ] Risks identified
- [ ] Ready for architect review
